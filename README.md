**The Dental Paradise — Mobile App UI for Appointment Booking ,Future Interns, UI/UX Design, Task 2**

**About The Project :**
This is my Task 2 submission for the Future Interns UI/UX Design Internship. I designed and built a mobile app UI for an appointment booking system for a dental clinic called The Dental Paradise based in Hyderabad, India.

The main focus was on creating a smooth and intuitive booking flow that feels trustworthy and easy to use for patients, while also looking clean and professional as a healthcare app.

**Screens Built :**
Onboarding Screen — Custom tooth logo, brand identity and get started flow.
Login Screen — Email, password fields and social login options.
Home Screen — Greeting, upcoming appointment banner, services, doctor listings and live appointments section.
Specialist Selection Screen — Doctor cards with filter chips, star ratings and availability badges.
Booking Screen — Weekly calendar strip, date selection and time slot booking interface.
Confirmation Screen — Success animation and full booking summary card.

**Features :**
Dynamic doctor routing so each doctor's booking page shows the correct doctor details.
Real time appointments section that updates on the home screen after every confirmed booking.
Time slot selection with available, selected and unavailable states.
Weekly calendar strip with date selection.
Filter chips for specialist category selection.
Availability badges showing whether a doctor is available today or tomorrow.
Clear All button to reset booked appointments directly from the home screen.
Bottom navigation bar with Home, Appointments and Doctors tabs.
Custom tooth logo consistent across onboarding and login screens.

**Design Decisions :**
Sky Blue and Deep Navy were chosen as the primary colors to convey healthcare trust and professionalism.
Warm Amber was used for CTA buttons to add warmth for patients who may feel anxious about dental visits.
Mint Green was used for success states and availability badges to reinforce freshness and health.
Lora serif was used for headings to convey medical expertise and credibility.
DM Sans was used for body text to keep the UI clean and modern.

**Tools and Technologies :**
Figma AI was used for design and prototyping.
React.js was used for building the app.
TypeScript was used for type safety.
Tailwind CSS was used for styling.
React Router DOM was used for navigation between screens.
Lucide React was used for icons.
GitHub was used for storing the code.

**Project Structure :**
app folder contains the main screens and components.
imports folder contains the dependencies.
styles folder contains the styling and theme files.
